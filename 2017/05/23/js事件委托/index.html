js事件委托

发表于 2017-05-23

概述

事件委托也叫事件代理，它的原理就是利用事件冒泡机制，只指定一个事件处理程序，就可以管理某一类型的所有事件。

为什么要用事件委托

减少dom操作／减少对象占用内存

通常情况下，给dom添加事件处理程序，会直接给它设置一个事件处理程序，但假设有很多的dom需要添加事件处理程序，比如有100个li，每个li都有相同的click点击事件，通常做法事用for循环，遍历所有的li，然后给它们添加事件。

这种处理方式需要不断的与dom节点进行交互，操作多次dom节点，而所以会影响页面的整体运行性能。
用事件委托来处理的话 可以把事件处理绑定到li的父元素 ul上 这样与dom的操作就只需要一次，大大提高了性能。
1、访问dom的次数越多，引起浏览器重绘与重排的次数也就越多，就会延长整个页面的交互就绪时间，这就是为什么性能优化的主要思想之一就是减少DOM操作的原因
2、每个函数都是一个对象，是对象就会占用内存，对象越多，内存占用率就越大，100个事件处理程序就占用一百个内存空间，自然性能就差

事件委托原理

事件委托是利用事件的冒泡原理来实现的。就是避免对特定的每个节点添加事件监听器；而是委托它们父级代为执行事件。事件监听器添加到它们的父元素上。事件监听器会分析从子元素冒泡上来的事件，找到是哪个子元素的事件。
